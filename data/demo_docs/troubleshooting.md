# MallPilot 使用说明手册

## 商品推荐问题

如果推荐结果不够精准，建议优先补充以下信息：

1. 商品类目，例如手机、扫地机器人、电动牙刷。
2. 预算范围，例如 300 元内、1000-1500 元。
3. 使用场景，例如送礼、老人使用、通勤、居家。
4. 核心诉求，例如续航、静音、轻便、易清洁。

## 订单查询问题

如果需要查询订单，请优先提供订单号，例如 `MP20260706001`。系统会根据订单库展示：

- 当前订单状态
- 物流状态与物流公司
- 支付方式和实付金额
- 发票抬头和售后状态

## 售后咨询问题

如果需要说明退货、退款或换货流程，建议同时说明：

1. 订单号
2. 商品名称
3. 问题描述，例如瑕疵、颜色不符、不好用
4. 希望的处理方式，例如退货、换货、重新推荐替代款

## 规则说明问题

如果问题属于运费、会员积分、优惠券、发票或退换货政策，MallPilot 会优先从知识库返回规则说明。若同时命中商品或订单数据，系统会把结构化查询结果与规则说明一起提供，避免只给抽象政策。
